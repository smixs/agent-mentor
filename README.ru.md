# mentor

mentor это skill для AI-агентов: Claude Code, Codex и любых других, читающих формат SKILL.md. Skill читает твою локальную историю сессий и собирает один самодостаточный HTML-отчёт о том, как ты на самом деле работаешь: что делаешь, где теряешь время и какие правки сделают тебя быстрее. Это session-insights skill по образцу встроенной команды Claude Code `/insights`, только направленный на ту часть цикла, которую ты никогда не измеряешь, на себя.

Агенты теперь делают работу, которую раньше делали джуны. Но джуны становились сеньорами через обратную связь от сеньоров. Раздай агентов джунам и убери сеньоров, и этот контур обратной связи исчезнет. Этот skill его возвращает, упакованный в агентский skill, который ставишь один раз и гоняешь раз в неделю.

![Пример отчёта](assets/report.webp)

## Что делает skill

- Читает локальную историю сессий: транскрипты Claude Code в `~/.claude` и роллауты Codex в `~/.codex`. Ничего не выгружается, skill работает целиком на твоей машине.
- Агрегирует много сессий в сводные цифры (сообщения, сессии, вызовы инструментов, правки, затронутые файлы, токены) плюс активность по дням, набор инструментов и языки.
- Пишет один самодостаточный `report.html` из восьми секций с SVG-иконками, счётчиками и анимированным графиком. Без зависимостей и сборки, открывается в любом браузере.

| Секция | Что даёт |
|---|---|
| Над чем ты работаешь | Сессии, собранные в темы, с примерной долей |
| Как ты используешь агента | Соотношение инструментов, языки, тайеринг моделей, чекпоинты |
| Что получилось сильно | Реальные удачи, возвращённые тебе |
| Где всё идёт не так | Точки трения, у каждой факты и конкретный фикс |
| Возможности, которые стоит попробовать | То, что ты недоиспользуешь |
| Что добавить в CLAUDE.md | Готовые строки, которые убирают повторяющееся трение |
| Новые способы использования | Рабочие сценарии под твои привычки |
| Куда двигается твой воркфлоу | Куда идёт твоя работа |

Секция про трение это смысл скилла. Под каждой точкой стоит повторяемый фикс: строка в CLAUDE.md, хук или привычка, чтобы одна и та же ошибка перестала всплывать через неделю.

## Установка скилла

Рекомендуемый способ, через [skills.sh](https://skills.sh), открытый маркетплейс агентских скиллов. Одна команда ставит skill в любой поддерживаемый агент:

```bash
npx skills add smixs/mentor
```

Полезные флаги:

```bash
# глобально, для всех проектов
npx skills add smixs/mentor -g

# в конкретные агенты
npx skills add smixs/mentor -a claude-code -a codex

# без вопросов, для CI
npx skills add smixs/mentor -g -y
```

Обновить skill до свежей версии: `npx skills update mentor`.

Вручную для Claude Code skill клонируется прямо в папку скиллов:

```bash
git clone https://github.com/smixs/mentor.git ~/.claude/skills/mentor
```

Для Codex клонируй skill в `~/.codex/skills/mentor`. Встроенным скриптам нужен [`uv`](https://github.com/astral-sh/uv); сам skill это обычный Markdown, его читает любой агент со скиллами.

## Использование скилла

Запусти команду или попроси агента своими словами («разбери мои сессии», «оцени как я работаю с агентом»):

```
/mentor            # оба агента, последние 30 дней (по умолчанию)
/mentor claude     # только сессии Claude Code
/mentor codex      # только сессии OpenAI Codex
/mentor both       # сквозной разбор: привычки, общие для обоих инструментов
```

Окно задаётся через `--days N`. Skill пишет отчёт в `~/.claude/usage-data/mentor-report.html` и открывает его. Сделай skill привычкой раз в неделю: паттерны видно на серии прогонов, а не на одном.

```bash
claude -p "/mentor both"
```

## Как устроен skill

Три небольших скрипта, без магии:

1. `collect.py` парсит каждую сессию окна в один набор статистики. Цифры честно посчитаны по логам.
2. Агент читает этот дайджест и пишет анализ из восьми секций с опорой на твои реальные промпты и метрики.
3. `render_report.py` сливает статистику и анализ в финальный HTML.

Отчёт пишется на твоём языке, по умолчанию английский, русский chrome тоже встроен в skill. Часть оценок (темы, удовлетворённость) выведена моделью и помечена в отчёте, читай их как гипотезу.

## Приватность

Skill читает локальные логи и пишет локальный HTML. Твои транскрипты не покидают машину.
